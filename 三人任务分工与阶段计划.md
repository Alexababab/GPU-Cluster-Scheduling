### 3.1 成员 A 的任务

#### 数据模型

- 定义 `Server`：
  - 服务器编号；
  - GPU 数量；
  - 单卡显存；
  - CPU 核数；
  - 内存容量。
- 定义 `Task`：
  - 任务编号；
  - 提交时间；
  - 运行时长；
  - 最低 GPU 数量；
  - 总显存需求；
  - CPU 需求；
  - 内存需求；
  - 优先级权重。
- 定义 `Assignment`：
  - 任务编号；
  - 服务器编号；
  - 开始时间；
  - 实际 GPU 数量；
  - 完成时间。
- 定义完整调度结果的数据结构。

#### 输入与预处理

- 实现标准输入解析；
- 检查数据读取是否完整；
- 计算任务在每台服务器上的最小 GPU 数；
- 计算每个任务的可行服务器集合；
- 计算每个任务的可行服务器数量；
- 标记不可在某台服务器运行的任务；
- 编写输入解析单元测试。

#### 调度状态

- 实现服务器资源占用状态；
- 支持查询指定时间区间的 GPU、CPU、内存余量；
- 支持查找任务在服务器上的最早可行开始时间；
- 支持将任务加入调度状态；
- 保证资源更新与查询结果一致。

#### 第一版调度器

- 实现保证可行的简单贪心；
- 按固定顺序处理任务；
- 为任务选择最早可启动的服务器；
- 使用任务在目标服务器上的最小可行 GPU 数；
- 生成全部任务的调度结果；
- 按任务编号输出。

#### 交付物

- 核心数据结构；
- 输入解析模块；
- 调度状态模块；
- 最早可行时间查询；
- 第一版合法贪心调度器；
- 相关单元测试。

### 3.2 成员 B 的任务

#### 合法性验证器

- 检查输出行数；
- 检查任务编号范围；
- 检查任务是否恰好出现一次；
- 检查是否遗漏任务；
- 检查是否重复任务；
- 检查服务器编号是否合法；
- 检查开始时间不早于提交时间；
- 检查 GPU 分配数量为正整数；
- 检查 GPU 数量不低于最低需求；
- 检查 GPU 数量不超过服务器容量；
- 检查总显存需求能够装入分配的 GPU；
- 检查 CPU 和内存单任务可行性；
- 检查完成时间等于开始时间加运行时长；
- 检查任务运行期间服务器 GPU 不超限；
- 检查任务运行期间服务器 CPU 不超限；
- 检查任务运行期间服务器内存不超限；
- 输出明确的错误类型和任务编号；
- 验证器日志只能输出到标准错误或独立工具中。

#### 指标计算器

- 计算每个任务的等待时间；
- 计算加权等待时间 `E_wait`；
- 计算每个时段的 GPU 显存使用量；
- 计算 GPU 显存空闲指标 `E_memory`；
- 计算全部任务最晚完成时间 `E_finish`；
- 输出三个原始指标；
- 使用手工小样例验证计算结果。

#### 验证测试

- 制作遗漏任务样例；
- 制作重复任务样例；
- 制作开始时间非法样例；
- 制作完成时间非法样例；
- 制作 GPU 超限样例；
- 制作 CPU 超限样例；
- 制作内存超限样例；
- 制作显存不足样例；
- 制作时间区间重叠超限样例；
- 确认验证器能够识别所有故意构造的错误。

#### 交付物

- 独立合法性验证器；
- 三项指标计算器；
- 验证器手工测试集；
- 验证与评分结果输出格式。
