# V0 合法基线设计

## 目的

先建立可构建、可运行、输出完整且资源绝不冲突的闭环，为验证器、指标计算器和后续并行贪心提供固定参照。

## 当前策略

1. 按任务编号处理任务。
2. 对每台服务器计算最小可行 GPU 数：
   `max(min_gpu, ceil(total_gpu_memory / per_gpu_memory))`。
3. 选择所需 GPU 数最少的可行服务器，平局选择编号较小者。
4. 所有任务全局串行，开始时间为提交时间与上一任务完成时间的较大值。

## 正确性边界

- 每个任务输出一次；
- 开始时间不早于提交时间；
- 完成时间等于开始时间加运行时长；
- 单任务 GPU、显存、CPU、内存均满足目标服务器容量；
- 因任务全局串行，不存在并发资源超限。

## 下一步（角色 A）

将“全局串行”替换为按服务器维护的事件驱动资源状态，在不破坏合法性的前提下允许多个任务并发运行。
