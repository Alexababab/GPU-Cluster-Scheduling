# V0 Greedy 设计

## 目的

建立可构建、可运行、输出完整的并行贪心闭环，为后续任务排序、Filter + Score 和搜索算法提供稳定起点。

## 当前策略

1. 按提交时间、任务编号维护固定任务顺序。
2. 对每个任务和服务器预计算最小可行 GPU 数：
   `max(min_gpu, ceil(total_gpu_memory / per_gpu_memory))`。
3. 以任务提交和完成为事件推进离散时间。
4. 每个事件时刻扫描等待队列，启动所有当前可放置的任务。
5. 服务器选择采用 best-fit：依次最小化放置后的 GPU、CPU、内存余量，最后按服务器编号打破平局。
6. 完成事件释放 GPU、CPU 和内存资源。

## 正确性边界

- 每个任务输出一次；
- 开始时间不早于提交时间；
- 完成时间等于开始时间加运行时长；
- 单任务 GPU、显存、CPU、内存均满足目标服务器容量；
- 每次分配前检查实时剩余资源，完成时对称释放；
- 同一任务只能在一个服务器上连续执行。
