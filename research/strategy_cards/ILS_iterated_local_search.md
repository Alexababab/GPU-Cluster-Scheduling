# Iterated Local Search (ILS)

- 来源：Lourenço, H. R., Martin, O. C., & Stützle, T. (2003). Iterated Local Search. In *Handbook of Metaheuristics*.
- 原问题：旅行商问题（TSP）、二次分配问题（QAP）、流程车间调度（Flow Shop Scheduling）
- 核心机制：
  - 不重复随机重启，而是在**当前局部最优解**上做**扰动（perturbation）**，再重新局部搜索
  - 三层结构：(1) 初始解 → (2) 局部搜索 → (3) 扰动 → (4) 局部搜索 → (5) 接受准则 → 回到(3)
  - 扰动强度是关键参数：太弱容易回到原局部最优，太强退化成随机重启
  - 接受准则可接受劣化解（模拟退火风格），也可只接受更好解
  - 典型扰动：随机交换 k 个元素、破坏+重建（与 LNS 交叉）
- 与本题的差异：
  - 原问题是排列/路径优化（TSP 的排列解），本题是**任务-服务器-时间三维分配**
  - ILS 的"解"是完整排列，本题的"解"是每个任务的 server_id + start_time + gpu_count
  - 本题的合法解必须满足八项约束，扰动后可能直接非法，需要校验或修复
- 本题可采用部分：
  - **Perturbation 操作**：随机移除 q 个任务的调度，然后重新贪心插入（即 LNS 思路）
  - **接受准则**：SA 式概率接受，允许跳出局部最优
  - **两级扰动强度**：探索期强扰动（40-50% 任务重排），收敛期弱扰动（5-10%）
  - 可将贪心调度器作为"局部搜索"的内核
- 计划修改：
  - 实现 `perturb(schedule, degree)`：随机选择 p% 任务，删除后重新调度
  - 实现 ILS 主循环：初始解 → 扰动 → 贪心修复 → 接受/拒绝
  - 支持通过 `--iterations` 参数控制 ILS 迭代轮数
  - 支持两种接受策略：OnlyBetter / SA
- 预计改善指标：
  - E_finish（最晚完成时间）——通过扰动打破贪心的局部最优
  - E_wait（加权等待时间）——通过重新排序任务启动顺序
- 预计适用实例：
  - 中小规模（N ≤ 200, M ≤ 20），ILS 迭代次数可控
  - 贪心调度器明显留下长尾任务的实例
  - 任务间资源竞争激烈的实例
