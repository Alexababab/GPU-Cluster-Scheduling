# 可行性异构感知的服务器机会成本评分

- 来源：Deepak Narayanan, Keshav Santhanam, Fiodar Kazhamiaka, Amar Phanishayee, Matei Zaharia, *Heterogeneity-Aware Cluster Scheduling Policies for Deep Learning Workloads*, OSDI 2020（Gavel）。
- 原问题：在包含多种 GPU 的异构集群中，不同深度学习任务在不同加速器上的吞吐率差异明显。传统调度器忽略这种任务—设备匹配关系，容易把稀缺的高价值设备分给收益不高的任务，从而恶化平均完成时间、makespan 和集群吞吐。
- 核心机制：建立任务在不同加速器上的吞吐矩阵，以“有效吞吐率”统一表达任务从不同资源配置获得的收益；再将 FIFO、最短作业优先、最小 makespan、公平调度等策略写成异构感知的优化目标。策略层给出目标资源份额，执行层通过轮次调度逼近目标分配。
- 与本题的差异：论文允许抢占、迁移、时间片轮转和 GPU 空间共享，并假设任务在不同 GPU 上具有可测量或可估计的吞吐率；本题任务运行时长固定、不可抢占、不可迁移，只能在一台服务器上连续运行，且没有任务在各服务器上的真实性能数据，也不适合依赖 LP/MILP 求解器。
- 本题可采用部分：保留“显式建模任务—服务器异构关系”和“避免浪费稀缺服务器”的思想。用任务—服务器可行矩阵替代吞吐矩阵；根据任务的可行服务器数量衡量任务稀缺度，根据一台服务器能够承载的稀缺任务集合衡量服务器机会价值。放置普通任务时，惩罚占用只有少数任务能够使用的服务器。
- 计划修改：
  1. 预计算 `feasible[i][j]`、任务可行服务器数 `fit_count[i]` 和最小可行 GPU 数。
  2. 定义任务稀缺权重 `scarcity[i] = 1 / fit_count[i]`，对仅有 1～2 台可行服务器的任务额外加权。
  3. 对候选服务器 `j`，估计本次放置造成的机会成本：统计放置后在该服务器上暂时或永久失去可行性的等待任务，并累加其稀缺权重与优先级权重。
  4. 将服务器评分改为  
     `score = α·开始时间代价 + β·显存碎片 + γ·GPU/CPU/内存碎片 + δ·稀缺服务器机会成本`。
  5. 任务排序中加入稀缺度和等待时间，避免稀缺任务长期被普通任务阻塞。
  6. 第一版只扫描当前等待队列；若耗时可接受，再加入尚未提交任务的静态保护项。
- 预计改善指标：主要降低 `E_memory`，减少异构服务器上的显存与资源碎片；其次降低高权重稀缺任务的 `E_wait`；在资源竞争密集实例上可能降低 `E_finish`。
- 预计适用实例：服务器 GPU 数量、单卡显存、CPU 或内存差异较大；任务可行服务器数量分布不均；存在大量只能在少数服务器运行的贴边任务；提交时间密集、资源竞争明显的实例。
- 实现成本：中。已有可行服务器集合和最小 GPU 数预计算，主要新增等待任务机会成本统计与可配置评分项；无需外部优化求解器。
- 实现者：角色 A。
- 实验对照：
  1. V0：原始 best-fit。
  2. V1a：best-fit + 稀缺任务优先。
  3. V1b：best-fit + 服务器机会成本。
  4. V1c：稀缺任务优先 + 服务器机会成本。
  5. 对 `δ` 做消融与多组权重实验，分别比较全体实例、异构度高实例和稀缺任务多实例的 `E_wait`、`E_memory`、`E_finish` 与运行时间。
- 实验结果：本题尚未实验。原论文在其工作负载和系统模型下报告：异构感知策略在模拟集群中将平均作业完成时间最多改善 3.5 倍、makespan 改善 2.5 倍；这些数值不能直接视为本题的预期收益。
- 最终决定：保留并建议作为角色 A 的首个异构服务器评分策略实现。采用可行矩阵、稀缺度和机会成本评分；不采用论文中的 LP/MILP、抢占迁移、轮次调度和空间共享机制。
